# 🤖 Automatic Handler Index Generation

**Version:** 1.1.0+  
**Status:** ✅ Production Ready

---

## Overview

The CAP Handler Framework now automatically generates `handlers/index.ts` files for all your services during CDS bootstrap. No more manual imports!

### Before 😓

```typescript
// srv/my-service/handlers/index.ts (MANUAL)
import BooksHandler from './entities/BooksHandler';
import AuthorsHandler from './entities/AuthorsHandler';
import PublishersHandler from './entities/PublishersHandler';
// ... add more imports manually

export const HANDLER_CLASSES = [
  BooksHandler,
  AuthorsHandler,
  PublishersHandler,
  // ... add more manually
];
```

### After 🎉

```
✨ Just create the handler file - that's it!
```

The framework automatically:
1. Scans all `entities/` and `proxies/` directories
2. Finds all `*Handler.ts` files
3. Generates the `index.ts` with all imports
4. Updates on every server restart

---

## How It Works

### 1. CDS Plugin Hook

The `cds-plugin.js` hooks into CDS bootstrap:

```javascript
// cap-handler-framework/cds-plugin.js
cds.on('bootstrap', async () => {
  await autoGenerateHandlerIndices();
});
```

### 2. Auto-Discovery

For each service in `srv/`:
- Scans `srv/<service>/handlers/entities/`
- Scans `srv/<service>/handlers/proxies/`
- Finds all files ending with `Handler.ts` or `Handler.js`

### 3. Index Generation

Creates/updates `srv/<service>/handlers/index.ts`:

```typescript
/**
 * AUTO-GENERATED by @ctac/cap-handler-framework
 * 
 * Generated: 2026-03-12T10:37:27.709Z
 * Service: my-service
 * Handlers: 15 total (5 entities, 10 proxies)
 */

// Entity Handlers
import BooksHandler from './entities/BooksHandler';
import AuthorsHandler from './entities/AuthorsHandler';
// ... (sorted alphabetically)

// Proxy Handlers
import CustomersProxyHandler from './proxies/CustomersProxyHandler';
// ... (sorted alphabetically)

export const HANDLER_CLASSES = [
  // Entity handlers (5)
  AuthorsHandler,
  BooksHandler,
  
  // Proxy handlers (10)
  CustomersProxyHandler,
  // ...
];
```

## Usage

### Zero Configuration Required! ✨

Just structure your handlers correctly:

```
srv/
├── my-service/
│   ├── my-service.cds
│   ├── my-service.ts
│   └── handlers/
│       ├── index.ts          ← AUTO-GENERATED (don't edit!)
│       ├── entities/
│       │   ├── BooksHandler.ts
│       │   └── AuthorsHandler.ts
│       └── proxies/
│           └── CustomersProxyHandler.ts
```

### Adding a New Handler

**Old way:**
1. Create `MyNewHandler.ts`
2. Open `index.ts`
3. Add import statement
4. Add to `HANDLER_CLASSES` array

**New way:**
1. Create `MyNewHandler.ts`
2. ✨ **That's it!** Restart server and it's automatically registered

## Git Configuration

### Add to .gitignore

Since `index.ts` is auto-generated, add it to `.gitignore`:

```gitignore
# Auto-generated handler indices (regenerated by cap-handler-framework plugin)
srv/**/handlers/index.ts
```

**Why?**
- Prevents merge conflicts
- Always in sync with actual handlers
- Clean git history
- No manual maintenance

### First-Time Setup

If you already have `index.ts` files:

```bash
# Remove existing manual files
rm srv/*/handlers/index.ts

# Add to .gitignore
echo "srv/**/handlers/index.ts" >> .gitignore

# Commit the gitignore change
git add .gitignore
git commit -m "chore: auto-generate handler indices"

# Start server (auto-generates index.ts)
cds watch
```

## Naming Conventions

### ✅ Files That Are Detected

- `*Handler.ts` in `entities/` directory
- `*Handler.js` in `entities/` directory
- `*Handler.ts` in `proxies/` directory
- `*Handler.js` in `proxies/` directory

### ❌ Files That Are Ignored

- Files not ending with `Handler.ts` or `Handler.js`
- Files in subdirectories of `entities/` or `proxies/`
- Files in `utils/`, `operations/`, `factories/`, etc.

### Examples

```
✅ entities/BooksHandler.ts          → Detected
✅ entities/MyCustomHandler.ts       → Detected
✅ proxies/CustomersProxyHandler.ts  → Detected

❌ entities/BookService.ts           → Ignored (not *Handler.ts)
❌ entities/utils/Helper.ts          → Ignored (in subdirectory)
❌ utils/SequenceManager.ts          → Ignored (not in entities/proxies)
❌ operations/searchBooks.ts         → Ignored (operations are different)
```

## Troubleshooting

### Index.ts Not Generated?

**Check:**
1. Is `handlers/` directory present?
2. Are there files ending with `Handler.ts` in `entities/` or `proxies/`?
3. Is the server restarting? (Try `cds watch` again)

**Debug:**
```bash
# Check directory structure
ls -la srv/*/handlers/

# Check for handler files
find srv -name "*Handler.ts"

# Force regeneration by restarting
cds watch
```

### "Module not found" Error?

**Cause:** Index.ts wasn't generated before TypeScript compilation

**Solution:**
```bash
# Ensure index.ts exists
ls srv/*/handlers/index.ts

# If missing, restart once to generate
cds watch
```

### Wrong Handlers in Index?

**Cause:** Old files not deleted

**Solution:**
```bash
# Remove handler file
rm srv/my-service/handlers/entities/OldHandler.ts

# Restart server (auto-regenerates without old handler)
cds watch
```

## Migration Guide

### From Manual to Auto-Generated

**Step 1:** Backup current index.ts
```bash
cp srv/my-service/handlers/index.ts srv/my-service/handlers/index.ts.backup
```

**Step 2:** Delete manual index.ts
```bash
rm srv/my-service/handlers/index.ts
```

**Step 3:** Update .gitignore
```bash
echo "srv/**/handlers/index.ts" >> .gitignore
```

**Step 4:** Restart server
```bash
cds watch
```

**Step 5:** Verify
```bash
# Check auto-generated file
cat srv/my-service/handlers/index.ts

# Should start with:
# /**
#  * AUTO-GENERATED by @ctac/cap-handler-framework
#  * ...
```

**Step 6:** Test
```bash
# Ensure all handlers load
# Check console for:
# [my-service-handlers] [INFO] Loaded handler for Books
# [my-service-handlers] [INFO] Loaded handler for Authors
# ...
```

Works automatically for all services:

```
srv/
├── catalog-service/
│   └── handlers/
│       └── index.ts  ← Auto-generated
├── order-service/
│   └── handlers/
│       └── index.ts  ← Auto-generated
```

### Custom Service Names

Works with any service name:

```
srv/
├── my-custom-service/
│   └── handlers/
│       └── index.ts  ← Auto-generated (detects custom names)
```

### No Handlers?

If a service has no `handlers/` directory, it's simply skipped:

```
srv/
├── simple-service/
│   └── simple-service.cds  ← No handlers, no problem!
└── complex-service/
    └── handlers/
        └── index.ts  ← Only this one generated
```

## Performance

### Generation Time

- **Typical project (20 handlers):** < 50ms
- **Large project (100 handlers):** < 200ms
- **Runs only on bootstrap:** No runtime overhead

### File System Overhead

- **Synchronous I/O:** Uses `fs.readdirSync` (fast for small directories)
- **Minimal scanning:** Only `entities/` and `proxies/` directories
- **Cached by Node:** Files are read once per bootstrap

## Best Practices

### ✅ Do

- Let the framework generate `index.ts`
- Add `index.ts` to `.gitignore`
- Use consistent naming (`*Handler.ts`)
- Organize into `entities/` and `proxies/`

### ❌ Don't

- Edit `index.ts` manually (changes will be lost!)
- Commit `index.ts` to git
- Put handlers in subdirectories of `entities/`
- Rename files without restarting the server

## FAQ

**Q: Can I disable auto-generation?**  
A: No, but you can provide explicit `handlerClasses` to `registerHandlers()` if needed.

**Q: Does it work with JavaScript (.js) files?**  
A: Yes! Both `.ts` and `.js` files are supported.

**Q: What if I have handlers in other directories?**  
A: Only `entities/` and `proxies/` are scanned. Use explicit registration for custom locations.

**Q: Can I customize the generated file?**  
A: No, it's standardized for consistency. If you need custom logic, use explicit `handlerClasses`.

**Q: Does it work in production?**  
A: Yes! It runs during bootstrap, before any requests are handled.
